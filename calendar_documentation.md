Документация проекта «Календарь событий»

1. Название проекта

«Календарь событий» — настольное приложение для ведения личного календаря с возможностью записи и просмотра событий по датам.

2. Цель и актуальность проекта

Цель проекта — разработать простое настольное приложение, позволяющее пользователю фиксировать события  на конкретные даты и быстро просматривать их без необходимости подключения к интернету и регистрации учётной записи.
Актуальность. Большинство современных календарей — это мощные, но «тяжёлые» сервисы: они требуют учётную запись, подключение к сети и предоставляют десятки функций, многие из которых рядовому пользователю просто не нужны. Для человека, которому нужно лишь быстро записать пару дел на день, такой инструмент часто избыточен. Локальное офлайн-приложение с минимальным интерфейсом закрывает эту узкую, но реальную потребность, а также является хорошим учебным примером работы с GUI и файловым хранением данных на Python.

3. Задачи проекта

1. Спроектировать понятный графический интерфейс с виджетом календаря.
2. Реализовать добавление события для выбранной даты с проверкой ввода.
3. Реализовать просмотр списка событий за выбранную дату.
4. Обеспечить сохранение данных между запусками приложения.
5. Протестировать работу приложения по заранее составленному сценарию и подготовить отчёт.
6. Проанализировать существующие аналоги и сформулировать вывод о направлении дальнейшего развития проекта.

4. Функциональные требования

ФТ-1 Приложение отображает интерактивный календарь для выбора даты.
ФТ-2 Пользователь может ввести текстовое описание события в отдельное поле.
ФТ-3 По кнопке «Сохранить событие» описание привязывается к выбранной в календаре дате.
ФТ-4 При попытке сохранить пустое описание приложение выводит предупреждение и не сохраняет запись.
ФТ-5 На одну дату можно сохранить несколько событий.
ФТ-6 По кнопке «Показать события» приложение выводит все события выбранной даты в текстовое поле.
ФТ-7 Если на выбранную дату событий нет, выводится соответствующее сообщение.
ФТ-8 Все события сохраняются в файл `events.json` сразу после добавления.
ФТ-9 При запуске приложение автоматически загружает ранее сохранённые события из файла.

5. Нефункциональные требования

НФТ-1 Кроссплатформенность — работа на Windows, Linux и macOS везде, где установлен Python и Tkinter.
НФТ-2 Полностью офлайн-работа, подключение к интернету не требуется.
НФТ-3 Время отклика интерфейса — мгновенное, так как все операции выполняются локально.
НФТ-4 Интерфейс на русском языке, понятный пользователю без технической подготовки.
НФТ-5 Минимальные требования к ресурсам — приложение не использует базы данных и сторонние сервисы, только один JSON-файл.

6. Используемые технологии и аргументация выбора

Python 3 — выбран как язык, рекомендованный заданием курса; кроссплатформенный, с понятным синтаксисом и обширной стандартной библиотекой, что ускоряет разработку учебного проекта.
Tkinter — встроенный в стандартную библиотеку Python модуль для построения графического интерфейса. Не требует установки дополнительных пакетов и обеспечивает прямое выполнение требования задания о наличии GUI («пользователя в консоль не отправляем»).
tkcalendar — добавляет готовый виджет интерактивного календаря с выбором даты в формате `yyyy-mm-dd`. Использование готового компонента избавляет от необходимости писать собственную логику отображения дней месяца и существенно сокращает объём кода, оставляя фокус разработки на бизнес-логике приложения.
JSON — встроенный в Python модуль `json` обеспечивает простое и человекочитаемое хранение данных в формате «дата → список событий». Для объёма данных одного пользователя полноценная СУБД избыточна, поэтому выбран лёгкий файловый формат.

7. Дополнительное задание: сравнительный анализ существующих решений

1. Google Calendar — облачный календарь от Google, доступный через браузер и мобильные приложения.
2. Microsoft Outlook Calendar (бесплатная версия Outlook.com и платный Microsoft 365) — календарь от Microsoft, доступный в составе почтового клиента Outlook, через браузер и мобильные приложения.
